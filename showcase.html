Core Features

为什么选择 Tools Manager

从框架设计到交互体验，每一个细节都为开发者效率而生。

🧩

SPI 插件机制

基于 @Tool 注解 + Spring 自动扫描，新增工具只需实现接口、打上注解，零配置自动注册上线。

🪟

多窗口并行

工具窗口可多开，同一工具支持多实例。Pinia + markRaw 管理状态，defineAsyncComponent 按需加载。

📊

瀑布流仪表盘

CSS Grid 12 列网格 + dense 瀑布流填充，支持拖拽排序与鼠标调整大小，自由编排你的工作台。

🔗

多输入位联动

转换类工具支持在任意输入位直接输入，其他位置自动联动更新，告别单向转换的低效。

🔍

弹窗放大查看

JSON、Base64、文本编辑器支持弹窗全屏化查看，拖拽分割条自由调整输入输出区域比例。

💾

状态持久化

仪表盘布局和工具输入内容自动缓存到 localStorage，刷新页面不丢失，一键清除重置。

Journey

工具开发历程

从框架设计到工具打磨，每一步都凝聚思考。

🏗️ 框架设计
设计 SPI 插件机制，定义 ToolDefinition / ToolExecutor 接口，@Tool 注解自动扫描注册。
🪟 多窗口管理
Pinia store + markRaw 避免响应式包裹，defineAsyncComponent 异步加载，解决 [object Promise] 问题。
🔧 三大基础工具
JSON 格式化（参考 bejson.com 优化）、Base64 混合解码、时间戳双向转换（秒/毫秒同时展示）。
📊 瀑布流仪表盘
CSS Grid 12 列 + dense 瀑布流，HTML5 Drag 排序 + mousedown/mouseup 调整大小，布局缓存 + 一键清除。
📝 文本编辑器
随手记录工具，支持 .txt 导入导出、字符统计、弹窗放大查看。
💾 存储单位换算
参考 cunchu.bmcx.com，B~YB 八级联动，千分位逗号选项，智能数值格式化。
🧮 计算器 + 进制转换
计算器支持四则运算、幂运算、位运算；进制转换 2/8/10/16 互转，多输入位联动。
🔗 多输入位联动
转换类工具全面升级：时间戳、存储单位、进制转换均支持在任意输入位输入，其他位置自动联动更新。
🔍 弹窗放大 + 拖拽分割
JSON、Base64、文本编辑器支持弹窗全屏查看；useSplitter composable 实现拖拽分割条调整输入输出区域比例。
